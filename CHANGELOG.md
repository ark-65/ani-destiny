# AniDestiny 更新日志

> 中文更新日志 | [English Changelog](./CHANGELOG_en.md)

## [Unreleased]

### 🐛 修复
- 修复“运行诊断”、播放器内播放诊断和复制出的支持摘要会在用户后来切换全局数据源后，拿新的“应用所选数据源”去改写旧的播放快照上下文的问题；现在这层信息会随播放快照一起捕获，支持链路看到的是当时现场，而不是事后状态。
- 修复“运行诊断”的最近播放快照和设置页复制出的反馈摘要在全局所选数据源、所选播放源、实际播放源三者继续错位时，仍得让人自己回头对照顶部全局设置才能看懂现场的问题；现在这两处也会像播放器内诊断一样，只在确实有助于解释当前播放上下文时补出“应用所选数据源”。
- 修复播放器内“播放诊断”和复制出的播放诊断摘要在全局所选数据源、所选播放源、实际播放源三者发生错位时，仍缺少“应用所选数据源”这一层上下文的问题；现在 AniDestiny 只会在这层信息确实有助于解释现场时补出来，让支持链路既不漏关键背景，也不平白堆字段。
- 修复播放器内“播放诊断”和复制出的反馈摘要会因为用户只是打开或复制支持信息，就把“采集时间”刷新成一个更晚时刻的问题；现在播放器内诊断也会直接显示这次快照的采集时间，并与“运行诊断”、复制摘要共用同一份最近播放快照，让支持链路先讲真实现场，而不是查看动作本身。
- 修复播放器已经自然进入加载完成、播放中、暂停待播或失败等新状态后，“运行诊断”里的最近播放快照却还可能停留在更早状态、要等用户手动打开播放器内诊断才更新的问题；现在 AniDestiny 会在播放诊断状态真正变化时自动刷新这份快照，让设置页、复制出的反馈摘要和刚刚发生的播放现场继续保持同一步调。
- 修复播放器内“播放诊断”已经带出播放状态，但“运行诊断”和复制出的反馈摘要还没沿用同一份播放状态快照的问题；现在三处支持入口都会一起显示这次快照采集时的播放状态，让用户和支持方不用在不同页面之间猜测当时究竟是在加载、缓冲、播放还是失败。
- 修复“运行诊断”和复制出的反馈摘要顶部仍把“应用当前数据源”写得像这次实际播放源的问题；现在会明确标成“应用所选数据源”，和“所选播放源”“当前播放源”“播放源状态”彻底分开，让用户在支持链路里不再把全局设置误读成当前播放现场。
- 修复“运行诊断”和播放器内播放诊断仍把“当前数据源”“原始数据源”“数据源状态”混在一起、用户不容易分清是全局设置还是这次实际播放来源的问题；现在这组字段会明确区分“所选播放源”“当前播放源”“播放源状态”，让 fallback 现场先讲清这次播放到底落在哪条源上。
- 修复播放器在 fallback 场景下复制或查看播放诊断时，仍主要只堆出“原始数据源 / 当前数据源”两个字段、用户还得自己推断 AniDestiny 到底为何改线的问题；现在诊断会直接补上一条“数据源状态”，明确说明原来源暂时不可用、当前正改由哪条数据源继续播放，让支持信息先讲结果、再讲字段。
- 修复播放器已经落入“播放暂时失败”现场时，用户点下 `下一集` 但应用还只是在后台确认能不能切过去，错误卡片里的 `重试` 入口就先整块消失的问题；现在这条恢复路径会继续留在原位并暂时禁用，让失败现场在核验阶段保持稳定，不会过早像已经正式切场。
- 修复播放器在点击 `下一集` 后还只是后台确认“下一集能不能播”的阶段，就先把当前页当成已进入正式交接、过早锁死返回/退出全屏并收起当前 fallback 提示的问题；现在只有真正开始切集、打开外部播放器或重试接管后才会进入页面级忙碌态，让验证阶段继续保留当前观看上下文和稳定退出路径。
- 修复播放器点下 `下一集` 后还在后台确认下一集能不能播的阶段，就先把整页切进忙碌态、收起弹幕并抹掉当前进度显示的问题；现在 AniDestiny 会先保留当前画面与观看上下文，只让 `下一集` 自己进入校验中的禁用态，等真正确认可以切过去后再统一进入正式交接。
- 修复 `下一集` 在还没确认下一集真的有可播线路前，就会先打断当前播放或清掉失败卡片的问题；现在 AniDestiny 会先把这条路查实，只有确认能真正切过去后才进入切集忙碌态，让继续观看的承诺更稳、更像一次真实接管。
- 修复播放器已经明确知道“当前就是最后一集”时，失败卡片里的主要恢复按钮却仍写着 `下一集`、要等用户点完才发现这条路走不通的问题；现在这个按钮会直接改成 `最后一集`，同时保留原有说明提示，让失败现场在用户触达前就先把边界讲清楚。
- 修复 AniDestiny 在首页、搜索、番剧详情、追番时间表、历史记录等 fallback 提示里仍把结果说成“备用数据”的问题；现在会直接说明已经改为显示其他数据源的内容，让播放器外的 source fallback 也更像用户能理解的结果，而不是实现细节。
- 修复播放器在当前一集恰好处于暂停态时，用户主动点 `下一集` 仍会把新一集留在暂停状态、像“只跳页没继续看”的问题；现在 `下一集` 会被视为明确的继续观看动作，只要切换成功就会立刻开始播放，让切集链路更连贯。
- 修复播放器在“播放暂时失败”后成功 `重试` 时，观看记录仍可能因为控制器状态稍后才回流而短暂写回 `00:00` 或旧位置的问题；现在恢复链路会立即按这次恢复位点写回历史，让用户就算马上离开或稍后继续观看，进度也和刚恢复的播放现场保持一致。
- 修复播放器刚进入“播放暂时失败”或一次 `重试` 再次失败后，观看记录还可能停留在更早位置的问题；现在失败现场会立即把这次中断位点写回历史，让回到继续观看时看到的进度和失败页保持一致。
- 修复播放器一次 `重试` 失败后，会把刚才中断的播放位点和倍速上下文一起丢掉、让下一次重试更像“从头再来”的问题；现在即使这次重试没成功，失败页也会继续保留上一轮中断现场，让用户看到的状态和下一次重试都还是在接着刚才那次播放恢复。
- 修复播放器失败现场已经直接提供 `重试` 按钮时，错误提示和禁用控件文案却仍在劝用户“稍后重试”的问题；现在失败提示会直接改成“请重试或尝试其他播放线路”，让恢复路径和页面动作保持同一套语言。
- 修复播放器已经落入“播放暂时失败”现场时，错误卡片里的 `下一集` / `外部播放器` 动作仍不像主播放器控件那样直接解释边界的问题；现在这些恢复动作也会沿用同一套提示和弱化样式，让“已经是最后一集”或“当前线路暂时不能交给外部播放器”在失败现场同样一眼说清。
- 修复播放器已经落入“播放暂时失败”现场后，“继续看下一集”仍主要藏在顶部或全屏控件里、用户得先离开失败现场再找这条恢复路径的问题；现在错误卡片会直接提供 `下一集` 入口，并继续沿用既有的切集忙碌态与“已经是最后一集”解释，让失败后的继续观看也留在同一个现场完成。
- 修复播放器在“播放暂时失败”现场遇到受保护线路时，会把 `外部播放器` 这一条主要恢复动作直接藏掉、只剩一段解释小字的问题；现在失败卡片会继续保留这个入口并诚实禁用，让“这次不能交给别的播放器”在失败现场也和普通播放器页保持同一套表达。
- 修复播放器当前线路因为额外请求条件而暂时不能交给外部播放器时，页面提示仍只会泛化地说“这条播放线路”的问题；现在禁用提示、失败卡片和即时反馈都会直接点明实际留在 AniDestiny 里的播放源，让这条边界也和最近的 handoff 成败提示一样具体。
- 修复播放器把当前播放交给外部播放器成功或失败后，提示语仍只泛化地说“当前播放”而不点明实际播放源的问题；现在反馈会直接说出交接的是哪条数据源，让 fallback 之后的外部交接结果也一样具体、可信。
- 修复播放器已经改从备用数据源继续尝试、但最终仍落入“播放暂时失败”时，错误卡片第一屏仍只剩诊断式 source/line 信息的问题；现在会先像顶部 fallback 横幅一样直接说明 AniDestiny 已改从哪条数据源继续播放，再保留下方诊断上下文，让失败现场先讲清结果、再讲细节。
- 修复播放器成功切到下一集、但这次切换实际改走备用数据源后，底部提示仍只会泛化地说“当前数据源暂时不可用”的问题；现在会像播放器页横幅一样直接说明 AniDestiny 改用了哪条数据源继续播放，让切集后的结果更具体也更可信。
- 修复播放器当前其实已经平稳切到备用线路后，顶部提示仍用“备用播放数据”这类实现味说法的问题；现在会直接说明“已改用哪条线路继续播放”，让 source fallback 这件事更像用户能理解的结果，而不是内部机制。
- 修复播放器已经在全屏里进入“切换下一集 / 正在重试播放 / 正在打开外部播放器”等关键交接态时，屏幕上可见的 `退出全屏` 入口对触屏用户仍只是一个点了没反应的死按钮的问题；现在它会沿用当前忙碌原因直接解释“还不能离开”，让可见退出口也和系统返回、普通返回箭头保持一致。
- 修复播放器把当前播放交给外部播放器失败后，提示语仍只会泛化地让用户“稍后重试”、却没说明 AniDestiny 里这一集其实还留在原处的问题；现在失败提示会直接说明“当前播放会继续留在 AniDestiny”，让这次交接失败后的结果本身也足够诚实。
- 修复播放器已经进入“切换下一集 / 正在打开外部播放器 / 正在重试播放”等关键交接态时，离开页面仍只会统一提示“当前播放操作尚未完成”、却不说明到底在等什么的问题；现在返回箭头和系统返回都会按当前动作分别说明“等下一集加载完 / 等外部播放器打开完 / 等重试完成”后再离开，让退出提示也和页面上的忙碌态保持同一套诚实语言。
- 修复播放器已经进入“切换下一集 / 正在重试播放 / 正在打开外部播放器”等关键交接态时，普通页面左上角返回箭头会直接变成一个点了没反应的死按钮、而系统返回却还能明确提示“请稍候”的问题；现在这个返回入口也会沿用同一条忙碌说明，让用户在每个离开入口上拿到一致反馈。
- 修复播放器把当前线路成功交给外部播放器后，页面除了外部 App 本身几乎没有任何明确确认、让这次切换更像“也许点上了”的问题；现在交接完成后会补上一条简短确认提示，让“已经成功交给外部播放器”在 AniDestiny 内也被清楚说出来。
- 修复播放器已经进入“播放暂时失败”现场后，上一集弹幕和弹幕状态徽标还继续挂在失败页面上的问题；现在错误卡片出现时会先收起这些旧观看信息，让页面只表达当前失败和可恢复动作，不再把旧播放残影继续留在现场。
- 修复播放器在普通页面进入“切换下一集 / 正在重试播放 / 正在打开外部播放器”时，顶部只保留剧集名、却不把当前动作轻量说出来的问题；现在嵌入态 AppBar 会继续保留当前或即将接管的剧集标题，并补上一行简短状态，让等待中的页面既安静也更诚实。
- 修复播放器已经处在“播放暂时失败”现场时，用户点“下一集”但下一集最终没能真正打开，提示语还可能只落回“没有可播放线路”或“数据源暂时不可用”、没把“其实还留在当前这一集”说清楚的问题；现在只要切集没真正离开当前页，就会统一明确说明“下一集暂时无法打开，已保留当前这一集”，让失败后的停留结果本身也足够诚实。
- 修复播放器已经明确知道“这就是最后一集”时，`下一集` 入口只会静默不可用、触屏用户几乎拿不到任何原因的问题；现在嵌入态和全屏态都会继续用收敛过的样式保留该入口，并在用户触达时当场说明“已经是最后一集”，让这条边界不再像按钮失灵。
- 修复播放器从“播放暂时失败”页成功交给外部播放器后，会把当前失败现场一起清掉、导致用户再回到 AniDestiny 时像问题已经消失的问题；现在这类交接完成后会恢复原失败卡片，让“外部播放器能播”和“应用内这次没播起来”继续保持同一套诚实状态。
- 修复播放器已经进入“正在重试播放”后，全屏控制栏里的 `外部播放器` 入口仍像能立刻切走另一条交接路径的问题；现在它也会沿用同一套 `正在重试播放` 忙碌禁用态，避免恢复当前播放时又被第二个逃逸动作打断。
- 修复播放器明明已经掌握当前番剧详情、并且能判断“这就是最后一集”时，`下一集` 入口仍像可继续尝试的问题；现在嵌入态和全屏态都会收敛这个动作并明确提示“已经是最后一集”，避免用户先点一次、再被播放器事后告知这条路根本走不通。
- 修复播放器进入“切换下一集 / 正在重试播放 / 正在打开外部播放器”等关键交接态后，底部仍继续挂着上一条播放时间轴和进度条、像旧播放还在活着的问题；现在这些 route 级忙碌态会统一切回中性时间显示，只把当前正在发生的交接动作说清楚。
- 修复播放器在普通页面进入“切换下一集 / 正在重试播放 / 正在打开外部播放器”等关键交接态时，顶部标题里的状态行和中央过渡卡片会重复播报同一件事的问题；现在嵌入态只保留当前或即将播放的剧集标题，把动作说明集中交给中央过渡提示，让等待画面更安静。
- 修复播放器切“下一集”失败后，虽然实际上已经把用户留在当前这一集，提示语却还只会泛化地说“数据源暂时不可用”或“没有可播放线路”的问题；现在当切集失败但当前集仍被保留时，会直接说明“下一集暂时无法打开，已保留当前这一集”，让恢复结果本身也被诚实说清楚。
- 修复播放器在还没真正准备好播放、或者当前播放已经明确失败时，“进入全屏”仍像可立即使用的问题；现在只有当前线路真正可播时才会开放进入全屏，而已经在全屏里的用户仍可稳定退出，让这个动作更像观看控制而不是空承诺。
- 修复播放器当前线路依赖额外请求头、实际上根本不能安全交给外部播放器时，页面上的 `外部播放器` 入口仍像可点击、点下去才再弹一次同样限制说明的问题；现在入口会直接与提示文案保持一致地禁用，让“这次不能交给别的播放器”在用户触达前就足够诚实。
- 修复播放器在普通页面切到“下一集”后，一旦已经解析出目标剧集标题，顶部状态文案就会过早消失、只剩下新集标题，像是切换已经完成的问题；现在会继续保留简短的“正在切换下一集”状态，直到真正接管结束，让标题和页面忙碌态保持同一套诚实节奏。
- 修复播放器在普通页面进入“正在重试播放”或“正在打开外部播放器”等关键交接态时，旧的备用数据源提示横幅还会继续跟当前忙碌态争抢表达的问题；现在这些页面级交接会先收起 fallback 说明，让用户只看到当前正在进行的一件事。
- 修复播放器在当前线路其实无法直接交给外部播放器时，只会静默禁用按钮并继续抛出 `request headers` 这类实现味提示的问题；现在用户仍能点到明确原因，且说明改成产品语气，让“为什么这次不能交给别的播放器”更容易当场理解。
- 修复播放器在普通页面进入“切换下一集 / 正在重试播放 / 正在打开外部播放器”等关键交接态时，顶部标题不是直接丢掉剧集上下文、就是和中央过渡提示争抢表达的问题；现在嵌入态会继续保留当前或即将接管的剧集标题，只在需要时补一行简短状态，让过渡既诚实也更安静。
